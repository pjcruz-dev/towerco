TowerOS alignment summary

TowerOS is an enterprise multi-tenant SaaS (not an LLM agent product today). Auth splits Passport (central operators) and Sanctum (tenant users). Isolation uses stancl/tenancy (per-tenant MySQL databases). RBAC uses Spatie Permission in tenants and PlatformRoleCatalog + platform.permission middleware centrally.

  • Aligned: RBAC in application code (Rule 02), Eloquent/query bindings (05), per-tenant DB isolation (06), rate limits (08), audit logs for auth/billing/platform actions (09), subscription and operator kill switches (10).
  • Partial: Short-lived tokens (01 — tenant refresh rotation yes; platform Passport tokens are long-lived until rotation policy is added).
  • Not applicable yet: LLM prompt injection and untrusted model output (03–04) until AI features ship.
  • Hardening applied: P1/P2/P3 — platform RBAC + view permissions, dependency audits in CI, structured audit JSON to toweros-audit.log + stderr, platform operator TOTP MFA via TOWEROS_PLATFORM_MFA_REQUIRED, explicit CORS allowlist via config/cors.php + CorsAllowedOriginResolver (FRONTEND_APP_URL, tenant domains, TOWEROS_CORS_ALLOWED_ORIGINS).
